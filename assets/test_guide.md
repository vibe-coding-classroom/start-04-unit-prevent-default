# 移動端 UX 壓力測試指南 (Mobile UX Stress Test Guide)

在完成 **防止預設行為 (Prevent Default)** 的開發後，請依照以下步驟驗證你的 Web App 是否具備「原生感」。

## 1. 基礎鎖定測試 (Basic Lockdown)
- [ ] **長按測試**：在按鈕或背景長按 2 秒。
    - **失敗**：跳出系統選單（拷貝、查詢、分享）。
    - **成功**：什麼都沒發生，或是觸發你自定義的 UI 回饋。
- [ ] **雙擊測試**：快速雙擊螢幕任何地方。
    - **失敗**：頁面被放大 (Zoom in)。
    - **成功**：頁面保持原樣，操作依然流暢。

## 2. 邊界與回彈測試 (Edge & Rubber-banding)
- [ ] **拉扯測試**：在頁面最頂部或最底部猛力下拉/上滑。
    - **失敗**：看到瀏覽器背景色，或頁面產生「橡皮筋」回彈效果。
    - **成功**：頁面像固定在螢幕上，毫無動搖。
- [ ] **側邊滑動**：從螢幕邊緣向內滑動。
    - **注意**：現代手機（如 iOS/Android）的「返回手勢」通常難以完全封印，但應確保不會觸發頁面內的水平捲動。

## 3. 極限操作壓力測試 (Operational Stress Test)
- [ ] **多指暴力滑動**：使用 3 根手指在搖桿區快速、亂序地滑動。
    - **失敗**：頁面開始位移、縮放，或是搖桿反應遲鈍。
    - **成功**：`last-action` 狀態列精準更新座標，頁面結構穩如泰山。

## 4. 跨平台檢查點 (Cross-platform Checklist)
- **iOS (Safari)**: 檢查 `-webkit-touch-callout: none` 是否生效。
- **Android (Chrome)**: 檢查 `touch-action` 屬性在不同容器上的表現。
- **Modern Browsers**: 確保 JS 監聽器中正確設定了 `{ passive: false }`。

---
> 「原生感的關鍵，在於讓使用者忘記他正在使用瀏覽器。」
